權限

顯示目錄內所有檔案

ls [opiton] [path]

-a 顯示所有檔案包含隱藏檔

-l 顯示所有檔案的詳細資訊

檔案權限表示法: -rwxrwxrw- 第一個-代表檔案類型 d則為資料夾 然後後面每三碼分別代表user group other的權限,若顯示-則代表沒有該權限

更改檔案權限

chmod [number/symbol] [option] [filename/directory]

number 表示法 以3個8進位數字代表權限 000~777 r=4 w=2 x=1 三個相加等於該位置權限

3個八進位數字分別代表user group other

chmod 664 file 權限更改成-rw-rw-r--

symbol 表示法 以augo代表要更改的對象 然後用運算符號更改權限

chmod a+r file 全部人加上r的權限

chmod u=rwx,go=rx file user有全部權限 group other只有rx權限

chmod -R g=rx,o-w dir_name group對於資料夾有rx權限 group刪除w權限

權限遮罩

umask [number] 這邊只要記住一個觀念,umask就是建立一個新檔案時的權限遮罩,當使用者創立一個檔案時權限=666-umask 資料夾為777-umask 只是必須要換成符號表示法再計算才不會出錯

Bash shell

設定變數與字串概念

x=7 利用等號來設定變數

$x 利用$來取用變數

${x} 另外一種取用變數的方式

在shell中""包住的字串,仍然保有其特性(例如"$x" 會顯示7)

在shell中''包住的字串,失去其特性(例如'$x' 會顯示$x)

在雙引號中可以使用\來使單個特殊字元失去其特性(例如"\$" 會顯示$)

可以使用反單引號``或$小括號$()在雙引號包裹的字串中使用指令(例如 echo "i want `ls`或是 echo "i want $(ls)"")

萬用字元

? 長度為1的任意字元 (例如ls ?.txt 可以顯示目錄下 檔名長度1且副檔名為txt的檔案)

* 長度任意的任意字元(包含沒有字元) (例如ls *.txt 可以顯示目錄下任意長度且副檔名為txt的檔案)

[] 字元的集合,該位置的這個字元只能是中括號裡面的字元(例如 ls *[1-3].txt 會顯示任意長度且結尾為1~3且副檔名為txt的檔案)

[] 的寫法可以是範圍 [1-7][a-p][A-H] / 也可以每個元素都寫 [13579] 這樣就只會有偶數 / 也可以寫成[1-37-9] 這樣就是排除456

[^] [!] 兩種都是反選取,如[!1-7]就是排除1~7 [^1-37-9]就是只要456

尋找指令

這個指令跟ls不同的地方在於 他會從你給予的目錄底下搜索,且任何子目錄都搜索,並把搜尋條件所有的檔案所在的路徑+檔名都印出來,這時候搭配萬用字元非常好用

資料重新導向

>和>> 輸出重新導向, 將指令本來應該要輸出到螢幕上的東西,輸出到檔案或是其他指定的地方

<和<< 輸入重新導向, 觀念其實不好理解,因為目前所學能應用到符合邏輯的方法幾乎沒有,但是下面這個例子可以稍微幫助理解

wc > goo.txt < foo.txt 把foo.txt這個檔案 當作 wc 的輸入,並把回傳的結果,存到goo.txt裡面

wc < goo.txt > foo.txt 把goo.txt當作wc的輸入,並把結果輸出到foo.txt上

然後兩個連續兩個大於就是把結果直接輸出到檔案原本內容的後面,如果只有一個符號,則會覆蓋原本的結果 << 為結束輸入

最後 &這個符號,可以改變標準輸出和標準錯誤輸出的資料流, 例如echo "123" 這我們知道他一定是正確的 所以螢幕會印出123

所以我們在寫echo "123" 2>1.txt 的時候 1.txt不會有錯誤內容,因為她是對的 那我們改寫成 echo "123" 2>1.txt 1>&2 就是把標準輸出當作錯誤的,這樣前面2>1.txt 裡面就會有123

管線

| 為管線的符號 就是把左邊command的標準輸出 當作右邊command的標準輸入

如 ls -al | less 把ls -al的輸出 當作less的輸入

tee

可以把輸出的資料 同時輸出到檔案和螢幕 通常我們會配合管線做使用

如 ls -al | tee filename 把ls -al的輸出 分配到螢幕與檔案

我讀到這邊的時候有發現一個觀念問題, 為什麼不可以打 tee filename < ls -l 呢?

這邊要釐清一個觀念, 就是一般資料重新導向跟command重新導向是完全不一樣的 檔案就是用大小於符號 command就是用管線

管線你要根據左邊的指令(管線左邊不能是檔案 他是command的重新導向 給右邊的指令或資料)

但是資料重新導向則是相反,右邊必須要為資料,左邊則必須是command 這邊觀念非常複雜,一定要自己多嘗試才有辦法慢慢搞懂

/dev/null和/dev/tty

講到資料導向和標準輸出輸入就不得不提到這兩個資料夾

/dev/null很像windows的資源回收桶,但不完全是,因為丟進去的檔案室再也找不回來的,更類似於黑洞

而/dev/tty則是存放目前的控制終端
